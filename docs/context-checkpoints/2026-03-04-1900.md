## Architecture Reminders
- `createServiceClient()` = bypasses RLS | `createClient()` = user auth, respects RLS
- `allocation_status` enum: `pending | confirmed | active | completed | terminated | no_show`
- `ncr_severity` enum: `minor | major | critical`
- `allocations` columns: `status`, `agreed_day_rate`, `actual_start` (NOT `_date` variants)
- FK joins: `table!fk_name(cols)` required for ambiguous joins
- Next.js App Router: CANNOT pass regular functions Server→Client — use Server Actions or router.refresh()
- Telegram sessions: `phone_number = 'tg:{chatId}'` in `message_threads`
- Telegram callback data format: `action:param1:param2`
- `incident_time` and `staff_user_id` not yet in generated TypeScript types — use `as any` until regenerated
- Git email for Vercel deploy: must be `otatler@gmail.com`
- `ncr_comments` table not in TS types yet — use `(supabase as any).from('ncr_comments')`
